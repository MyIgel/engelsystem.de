Access Rights in the Engelsystem

The Engelsystem has a large amount of different access rights that gain the user access to specific functions. While assigning these rights directly to users would allow a very fine granulation of access rights it would be hard to maintain. To make it easier for the beaurocrats to assign the correct rights to users the Engelsystem uses a role model called "group rights". Such a group is a caollection of multiple rights that allow a user to fulfil a specific function. With a default installation of the Engelsystem the following group rights will be created:
  • 1-Gast
  • 2-Engel
  • Shirt-Manager
  • 3-Shift Coordinator
  • 4-Team Coordinator
  • 5-Bürokrat
  • 6-Developer

1-Gast

This is the guest role. It has the least amount of privileges and it's prupose is to give access to those functions needed to create a user in the Engelsystem (self-registration):

Rights gained from this role

  • register - Einen neuen Engel registerieren
  • login - Logindialog
  • start - Startseite für Gäste/Nicht eingeloggte User

2-Engel

This is the group all registered users are assigned to. It contains all rights required to be able to be an Angel.

Rights gained from this role

  • user_angeltypes - Join angeltypes.
  • angeltypes - View angeltypes
  • shifts_json_export - Export shifts in JSON format
  • atom - Atom news export
  • ical - iCal shift export
  • user_myshifts - Allow angels to view their own shifts and cancel them.
  • user_shifts - Signup for shifts
  • user_meetings - Lists meetings (news)
  • news_comments - User can comment news
  • user_questions - Let users ask questions
  • user_messages - Writing and reading messages from user to user
  • user_settings - User profile settings
  • logout - User darf sich ausloggen
  • news - Anzeigen der News-Seite

Shirt Manager

Shirt Managers are those users that hand out the T-Shirts that helpers get to thank them for their work. They have additonal right to see, whether another user is eligible to get a shirt and set a marker if the helper has already received the shirt.

Rights gained from this role

  • admin_active - Mark angels as active and if they got a t-shirt.

3-Shift Coordinator

Shift coordinators are those helpers trying to organize the other heplers. On a chaos event these are those angels that are located in heaven behind heavens desk. They setup shifts, news and meetings in the Engelsystem.

Rights gained from this role

  • admin_event_config - Allow editing event config
  • admin_free - Show a list of free/unemployed angels.
  • admin_shifts - Create shifts
  • admin_arrive - Mark angels when they arrive.
  • user_shifts_admin - Signup other angels for shifts.
  • admin_log - Display recent changes
  • admin_user - Administrate the angels
  • admin_news - Administrate the news section
  • register - Einen neuen Engel registerieren

4-Team Coordinator

Team Coordinators organize groups of angels that do very specific tasks. They can give angels access to restricted angel types and create news and meetings to announce introductions required to join restricted angeltypes.

Rights gained from this role

  • admin_user_angeltypes - Confirm restricted angel types
  • user_shifts_admin - Signup other angels for shifts.
  • admin_news - Administrate the news section

5-Bürokrat

Bureaucrats have permissions to create most of the event setup to enable other functions. They setup the rooms, angeltypes, shifttypes and other basic information for the event. Additionally they have the right to assign the access rights to others.

Rights gained from this role

  • shifttypes - Administrate shift types
  • admin_user_angeltypes - Confirm restricted angel types
  • admin_active - Mark angels as active and if they got a t-shirt.
  • admin_shifts - Create shifts
  • admin_import - Import rooms and shifts from pentabarf
  • admin_user - Administrate the angels
  • admin_news - Administrate the news section
  • admin_faq - Edit FAQs
  • admin_questions - Answer user''s questions
  • admin_angel_types - Engel Typen administrieren
  • admin_rooms - Räume administrieren
  • register - Einen neuen Engel registerieren

6-Developer

Developers have access to some very special functions, like editing the grouprights describes in this document, or bulk import data.

Rights gained from this role

  • admin_import - Import rooms and shifts from pentabarf
  • admin_language - Translate the system
  • admin_groups - Manage usergroups and their rights
